67  // Для работы со стандартными потоками IO нужно установить локаль с кодовой страницей, как у консоли.
68  // По умолчанию используется локаль "C" - русские буквы вообще не выводятся
69  // locale loc(""); создать локаль со страницей по умолчанию
70  // locale loc("Russian_Russia.1251"); создать локаль со страницей ANSI 1251
71  // locale loc("Russian_Russia.866"); создать локаль со страницей ОЕМ 866
72  // По умолчанию в консоли Windows используется страница ОЕМ 866
73  // Даже после установки локали с кодовой страницей ОЕМ 866
74  // для cout и cin, страница при выводе будет 1251 {cout и cin не учитывают кодовую страницу при выводе}
75  // для wcout и wcin, все работает нормально страница при выводе будет ОЕМ 866
76  // В следствии этого помимо смены локали на 866, нужно перегрузить оператор operator<< и operator>>
77  // Далее мы поменяем локаль на ОЕМ 866, но тут возникает еще одна проблема,
78  // руская локаль "Russian_Russia.866", не корректно выводит символы в шестнадцатиричном формате.
79  // Для решения этого комплекса проблем составим такую локаль, которая бралабы отображение цифр
80  // из локали "C", а все остальные настройки из локали "Russian_Russia.866".
81  // Примечание: Числа в 16-ом формате выводятся в локале "Russian_Russia.866",
82  // с группировкой по три цифры. 0xF0F0F0F0 -> F0 F0F 0F0
83  // Примечание: Строка с именем локали имеет следующий формат:
84  // язык_страна.codepage
85  // все три составляющии (язык,страна,codepage) должны
86  // быть корректными, но они не зависят друг от друга
87  // т.е. можно задать такое:
88  // "Russian_United Kingdom.10007"
